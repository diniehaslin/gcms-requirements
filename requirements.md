GREENCYCLE MANAGEMENT SYSTEM REQUIREMENTS DOCUMENT

TEN (10) FUNCTIONAL REQUIREMENTS (FR):
FR1: The system shall allow Staff and Admin users to log in securely using their credentials. 
FR2: The system shall allow Staff or Admin users to register new waste suppliers (individuals or companies). 
FR3: The system shall enable Staff users to record incoming waste, including weight (kg) and category (Plastic, Paper, Metal, Glass). 
FR4: The system shall automatically calculate the payout amount for suppliers based on current waste category rates. 
FR5: The system shall generate digital receipts for every successful waste transaction. FR6: The system shall allow Admin users to update the status of waste processing (Pending, Processing, Completed). 
FR7: The system shall track the current inventory of processed and unprocessed materials in the warehouse. 
FR8: The system shall allow Admin users to generate monthly waste processing and financial reports. 
FR9: The system shall display a real-time dashboard showing the total weight of waste collected today. 
FR10: The system shall allow Admin users to update the buying rates (price per kg) for different waste categories.

FIVE (5) NON-FUNCTIONAL REQUIREMENTS (NFR):
NFR1 (Security): All user passwords must be securely hashed before being stored in the database. 
NFR2 (Performance): The real-time dashboard must load within 2 seconds under normal network conditions. 
NFR3 (Availability): The system shall guarantee a 99.9% uptime during operational business hours. 
NFR4 (Usability): The web application interface must be responsive and accessible via both desktop computers and mobile tablets. 
NFR5 (Scalability): The database must be able to handle up to 5,000 transaction records per month without performance degradation.
